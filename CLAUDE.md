```sh
npm install         # install deps
npm run dev         # localhost:4321
npm run build       # production build to ./dist + Vercel adapter output to .vercel/output
```

## Environment

`PUBLIC_SITE_URL` must be set before deploy — consumed by `astro.config.mjs` for `site`. Without it, sitemap, canonicals, and all `og:*` tags point at the placeholder `https://tudominio.com`. Copy `.env.example` → `.env` in local.

## Architecture

Astro 6 with **hybrid rendering**: `output: 'server'` + `@astrojs/vercel` adapter. Every page marks `export const prerender = true` so the build emits static HTML; only `src/pages/api/og.ts` runs as a Node lambda. This keeps the door open for future SSR (API routes, form endpoints) without reconfiguring globally.

Integrations: `@astrojs/sitemap`, `astro-icon` (with `@iconify-json/lucide` + `@iconify-json/simple-icons` + `@iconify-json/flag`), `astro-compress`, `@tailwindcss/vite`. `prefetch.defaultStrategy: 'viewport'` (not `hover` — avoids wasting bandwidth on mobile). React was removed: no `.tsx`/`.jsx` files, no `client:*` directives — zero React consumers in src.

### Layout, routing, SEO

- [src/layouts/Layout.astro](src/layouts/Layout.astro) is the single shell. Owns `<head>` (SEO meta, Open Graph, Twitter Cards, favicons, JSON-LD `Person` + `WebSite`), loads Geist + Roboto Mono, injects the anti-FOUC theme script, renders `<Header />`, `<MobileDrawer />` (outside the header on purpose — see note below), `<main>`, `<Footer />`, and fires `setupSectionReveal` on `astro:page-load`. Exposes a `<slot name="head" />` for per-page JSON-LD (used by `BreadcrumbList` in case studies).
- Pages:
  - [index.astro](src/pages/index.astro) composes Hero → Projects (first 4) → About → Experience → Skills → Education. Contact lives on its own page (`/contact`).
  - [projects/index.astro](src/pages/projects/index.astro) lists **all** projects; the homepage shows just the top 4 plus a "Ver todos los proyectos" link here.
  - [404.astro](src/pages/404.astro) has `noindex` + lists up to 3 featured projects.
  - [api/og.ts](src/pages/api/og.ts) is the only SSR route. Generates 1200×630 PNG via `satori` + `sharp`, downloads Inter (Bold) TTF from jsDelivr on cold start, caches in memory. Fallback: 302 to `/og-image.png`.
  - `projects/[slug].astro`, `education/[slug].astro`, `certifications/[slug].astro` use `getStaticPaths()` over the Content Collections in `src/content/` to pre-render one page per entry. All share [CaseStudyHeader](src/components/CaseStudyHeader.astro) and render the Markdown body with the `.case-study-prose` styles in `global.css`.
- Homepage sections live in [src/components/sections/](src/components/sections/). Shared chrome (Header, Footer, Nav, MobileDrawer) in [src/components/layout/](src/components/layout/). Primitives (Button, Card, SectionTitle, BurgerButton, ThemeToggle) in [src/components/ui/](src/components/ui/).
- **Content authoring is Markdown-first**: each project, education item and certification is a `.md` file under [src/content/](src/content/) (`projects/`, `education/`, `certifications/`). Frontmatter validated by Zod in [src/content.config.ts](src/content.config.ts); body is the case study. Routes `/projects/<slug>`, `/education/<slug>`, `/certifications/<slug>` are generated by `getStaticPaths` + `getCollection`. The `entry.id` (file name without extension) is the slug.
- Home sections `Projects` and `Education` use `getCollection()` + `order` frontmatter to control display order.
- `src/data/` keeps three TS arrays that don't need Markdown: `skills.ts`, `experience.ts`, `navigation.ts`.

### Theme system

- Tokens as CSS custom properties in [src/styles/global.css](src/styles/global.css), both `:root` and `.dark` — OKLCH for neutrals with a subtle blue hue that matches the `#3846F5` logo. Tailwind v4 `@theme inline` maps them to utilities (`bg-background`, `text-foreground`, `bg-primary`, `bg-accent`, `border-border`, `ring-ring`, etc.). Add a new token by declaring it in both blocks + in `@theme inline`.
- [src/lib/theme.ts](src/lib/theme.ts): `getStoredTheme`, `setTheme`, `toggleTheme`, `applyTheme`, `bindThemeAcrossNavigations`. Dark default, no `prefers-color-scheme` fallback (the user's explicit preference wins; no preference means dark). Persists in `localStorage['theme']`.
- [src/components/ui/ThemeToggle.astro](src/components/ui/ThemeToggle.astro) is a delegated click handler keyed off `#theme-toggle`. Guard `window.__themeToggleBound` prevents duplicate listeners since the Header has `transition:persist`.
- Anti-FOUC: inline `<script is:inline>` in `<head>` applies `.dark` before first paint.

### Mobile drawer — why it lives in `<body>`, not `<header>`

The header uses `backdrop-blur-lg`, and `backdrop-filter` creates a **containing block for `position: fixed` descendants** (CSS Transforms 2 spec). When the drawer was nested inside the header, its `inset: 0` resolved to the header's 4rem-tall box instead of the viewport — so the panel slid off-screen invisibly. Fix: render [MobileDrawer.astro](src/components/layout/MobileDrawer.astro) directly inside `<body>` in Layout. State lives on `<html data-drawer-open>`, not component state, so it survives `astro:after-swap` and event delegation handles the burger click anywhere.

Focus trap, scroll lock, Esc-to-close, click-outside, and automatic close on resize-to-desktop are all in the drawer's inline `<script>`.

### Animations

`motion` (v12, renamed framer-motion DOM package) is imported **dynamically** — `await import('motion')` sólo se ejecuta si hay elementos animables en la ruta. Se usa en dos sitios:

- [src/lib/animations.ts](src/lib/animations.ts) — `setupSectionReveal()` anima nodos `[data-reveal]` y `[data-reveal-stagger]` con `inView + animate + stagger`.
- [src/components/sections/Hero.astro](src/components/sections/Hero.astro) — script inline que divide el H1 en letras (`<span class="hero-char">`) y las revela con stagger 22 ms (typewriter). Cursor pipe con blink CSS que se desvanece al terminar.

Animations respect `prefers-reduced-motion: reduce` — elementos saltan al estado final sin animar. Guards `__heroTypewriter` y `data-revealed="true"` evitan repetir animaciones entre View Transitions SPA.

### Navigation: active states + scroll-spy

[src/components/layout/Nav.astro](src/components/layout/Nav.astro) implementa un underline animado en `primary` que aparece en hover, focus-visible y active. En la home, un `IntersectionObserver` detecta qué sección está en el 40-55 % del viewport y marca el link correspondiente con `data-section-active="true"` — el mismo underline sirve para ambos estados. El scroll-spy sólo corre en `/` (las otras rutas no tienen las secciones ancladas).

El header usa `transition:persist` pero **no** `transition:name`: la animación por nombre causaba un salto visual en navegaciones SPA.

### View Transitions en imágenes de proyectos

[src/components/ProjectsGrid.astro](src/components/ProjectsGrid.astro) acepta `sharedTransition?: boolean`. Cuando está activo, cada `<img>` recibe `transition:name="project-media-<entry.id>"`. [CaseStudyHeader](src/components/CaseStudyHeader.astro) propaga ese nombre a la imagen principal del case study. Resultado: morph fluido home → `/projects/[slug]` y `/projects` → `/projects/[slug]`. Tanto el grid de la home como `/projects` lo pasan en `true`, así la navegación cruzada también encaja.

### Path aliases

Configured in [tsconfig.json](tsconfig.json) — prefer these over relative imports:

```
@/*           → src/*           (comodín, cubre data/lib/types/etc.)
@components/* → src/components/*
@layouts/*    → src/layouts/*
@styles/*     → src/styles/*
```

Alias específicos para `pages`, `assets`, `lib`, `data`, `types` fueron retirados por no tener consumidores; si los necesitas, añádelos de nuevo en `tsconfig.json`. Content lives in `src/content/{projects,education,certifications}/` as `.md` files loaded via `glob` loader (Content Layer API). Real images live in `public/images/{projects,education,certifications}/` and PDFs en `public/certifications/`.

### PDF previews — pre-render en build

PDFs subidos a `public/certifications/*.pdf` se convierten automáticamente a PNG (1700 px) en `public/images/certifications/` por el script [scripts/generate-pdf-previews.mjs](scripts/generate-pdf-previews.mjs) (usa `pdfjs-dist` + `@napi-rs/canvas`). Los hooks `predev` y `prebuild` en `package.json` lo ejecutan antes de cada `npm run dev` / `npm run build`. El script re-genera sólo si el PDF es más reciente que el PNG existente.

**Importante:** el script pasa `standardFontDataUrl` apuntando a `node_modules/pdfjs-dist/standard_fonts/` y `useSystemFonts: false`. Sin esto, PDFs con fuentes estándar (Helvetica, Times…) renderizan glifos vacíos en Node.

[src/components/PdfPreview.astro](src/components/PdfPreview.astro) detecta en build time si existe el PNG correspondiente al PDF y prioriza servirlo como `<img>` estático; si no, cae a render cliente con pdf.js.

### Certifications: image vs PDF

Cada certificación puede definir su preview como imagen o PDF:

- **Image** (`image:` en frontmatter): se muestra tal cual.
- **PDF** (`pdf:` en frontmatter): el script de previews lo convierte a PNG (mismo basename) en build time. [PdfPreview.astro](src/components/PdfPreview.astro) chequea con `fs.existsSync` si el PNG existe y lo sirve como imagen estática — carga instantánea, sin pdf.js en cliente. Si por alguna razón el PNG no existe (PDF recién añadido sin correr `npm run previews`), hace fallback a render cliente con pdf.js.
- **Ambos**: prioriza imagen, link "Ver PDF" en la caption.

### Prose styling for case study bodies

Markdown body of each entry renders inside `.case-study-prose` ([src/styles/global.css](src/styles/global.css)). This is a hand-rolled prose CSS (no `@tailwindcss/typography`): custom counters for `<ol>`, bullet dots for `<ul>`, code blocks with tokens, and `text-wrap: balance/pretty` on headings/paragraphs.

### Styling conventions

- Tailwind v4 utilities preferred over arbitrary values. Use tokens (`bg-primary`, `text-muted`) instead of raw palette (`bg-blue-500`, `text-slate-400`).
- Typography uses the fluid utilities in `global.css`: `.text-display-1`, `.text-display-2`, `.text-eyebrow`, `.text-lede`. They use `clamp()` for fluid scaling — avoid hardcoded `text-6xl` on H1.
- `cn()` helper ([src/lib/utils.ts](src/lib/utils.ts)) for conditional class composition. It's clsx + tailwind-merge.

### Icons

- `astro-icon` with `@iconify-json/lucide` (UI) and `@iconify-json/simple-icons` (brand logos).
- Import: `import { Icon } from 'astro-icon/components'`. Names: `lucide:mail`, `simple-icons:github`, etc.

### SEO

[Layout.astro](src/layouts/Layout.astro) acepta `ogType` y `article: { publishedTime, modifiedTime, author, section, tags }` para que los case studies emitan `og:type="article"` + `article:*` metas.

JSON-LD por tipo (además del global `Person` + `WebSite`):

- Projects `[slug]` → `CreativeWork` con `author`, `keywords`, `dateCreated`, `publisher`, `image`.
- Education `[slug]` → `EducationalOccupationalProgram` con `provider` (EducationalOrganization).
- Certifications `[slug]` → `EducationalOccupationalCredential` con `recognizedBy`, `credentialCategory`, `sameAs`.
- Todos los `[slug]` → `BreadcrumbList`.

Adicional: robots con `max-image-preview:large, max-snippet:-1`, `hreflang="es-CO"` + `x-default`, `format-detection="telephone=no"`. No se usa `astro-seo` — el SEO es manual para tener control total.
